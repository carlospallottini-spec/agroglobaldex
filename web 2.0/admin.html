Admin avanzado

Acciones operativas que NO van por el wizard de tokenizar. Distintos roles tienen distintos permisos on-chain — el programa rechaza las que no corresponden.
⚠️ Estas acciones modifican estado on-chain de forma inmediata. Devnet only mientras el audit + CASP estén pendientes.

1. Revoke KYC compliance_signer

Revoca KYC de una wallet ante sanctions hit, fraude, o requerimiento regulatorio. La wallet revocada no podrá comprar ni recibir tokens. NO afecta tokens ya en mano (para clawback hace falta PermanentDelegate, roadmap).
Emite ComplianceRevoked event con timestamp. Off-chain surveillance escucha y prioriza el caso.

2. Settle Investment Offering issuer

Registra on-chain el receipt de un yield distribuido off-chain (bank wire). Por epoch. Solo aplica a AssetClass::InvestmentOffering (e.g. Viñedo Rioja).
Se sube el SHA-256 (no el archivo). Guardá el original off-chain.

3. Update metadata (pre-mint) issuer

Actualizá nombre, metadata_uri y white_paper_uri antes de hacer el primer mint_token. Después del primer mint queda inmutable (revertirá con MetadataFrozen).

4. Transfer issuer current issuer

Cedé el rol issuer del registry a otra wallet. La nueva wallet DEBE tener ComplianceRecord KYC-verified (sin esto la ix revierte). Útil cuando vendés el SPV o rotás keypair operativo.
